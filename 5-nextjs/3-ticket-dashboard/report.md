# SEO Audit Report — Ticket Dashboard (Next.js 16 / App Router)

**Project:** `3-ticket-dashboard`
**Stack:** Next.js 16.2.4, React 19.2.4, Tailwind v4, MongoDB/Mongoose
**Audit Date:** 2026-05-02

---

## 1. Executive Summary

### Overall SEO Grade: **F (28/100)**

This project is, in its current state, effectively invisible and unindexable by search engines in any meaningful way. While the application is an internal-style **ticket management dashboard** (and arguably should not be publicly indexed at all), it ships with **zero SEO hygiene**: default boilerplate metadata, no `robots.txt`, no `sitemap.ts`, no `manifest.ts`, no canonical URLs, no Open Graph or Twitter cards, no structured data, an empty `next.config.ts`, and a critical `lang` attribute mismatch (declared `en`, content is Turkish).

Two strategic paths exist depending on intent:

1. **If this is a private/internal tool** (most likely): the priority is to **explicitly de-index** the entire site (robots.txt `Disallow: /`, `X-Robots-Tag: noindex`, meta robots `noindex, nofollow`) and add an honest, branded title for browser tabs. SEO becomes a leak-prevention task, not an acquisition task.
2. **If this is a public-facing product surface**: near-total metadata, structured-data, sitemap, manifest, OG/Twitter, i18n, and performance work is required before any organic visibility is possible.

The codebase should be brought to a defensible baseline regardless of indexing intent.

## 2. Findings by Category

Severity legend: **Critical** (blocks/wrongs indexing), **High** (major visibility loss), **Medium** (best-practice gap), **Low** (polish).

### 2.1 Metadata

#### F-001 — Default boilerplate title and description **[Critical]**
- **Where:** `src/app/layout.tsx` lines 17–20
- **What:** `title: "Create Next App"`, `description: "Generated by create next app"`.
- **Why it matters:** This is what shows in browser tabs, bookmark labels, social shares, AI chat citations, and (if indexed) Google SERPs. Unbranded and untrustworthy; AI assistants crawling the page will cite it as "Create Next App."
- **Fix:** Replace with a branded `title.template` (e.g., `"%s | Rudder"`) and `title.default` (e.g., `"Rudder — Ticket Yönetim Paneli"`) plus a meaningful Turkish description.

#### F-002 — No `metadataBase` defined **[High]**
- **Where:** `src/app/layout.tsx` (missing field)
- **What:** `metadataBase: URL` is not set.
- **Why it matters:** Without it, Next.js cannot resolve relative URLs for OG images, canonicals, alternates; emits a build-time warning; social previews break.
- **Fix:** Add `metadataBase: new URL(process.env.NEXT_PUBLIC_SITE_URL ?? "https://your-domain.tld")` in root layout metadata.

#### F-003 — No per-route metadata exports **[High]**
- **Where:** `src/app/page.tsx`, `src/app/tickets/page.tsx`, `src/app/ticket/[mode]/page.tsx`
- **What:** None of the routes export `metadata` or `generateMetadata`. Every route inherits the (broken) root metadata, producing identical title/description.
- **Why it matters:** Duplicate titles signal low quality; pages may be collapsed into one canonical SERP result; bad UX in tabs/history.
- **Fix:**
  - `/` → `metadata = { title: "Dashboard", description: "Ticket istatistikleri, dağılım grafikleri ve zaman bazlı analizler." }`
  - `/tickets` → `metadata = { title: "Tüm Ticket'lar", description: "Tüm ticket'ları kategori bazında görüntüleyin ve yönetin." }`
  - `/ticket/[mode]` → `generateMetadata` switching between `"Yeni Ticket Oluştur"` and `"Ticket'ı Güncelle: <title>"`.

#### F-004 — No canonical URLs **[High]**
- **Where:** All routes; `metadata.alternates.canonical` never set.
- **Why it matters:** Query-string variants and trailing-slash variants get indexed as duplicates, splitting authority. `/ticket/[mode]` is especially vulnerable since `mode` can be `add` or any ObjectId.
- **Fix:** Set `alternates.canonical` per route, statically or dynamically. For ID-based pages, normalize to the resolved ID URL.

#### F-005 — No `viewport` export (Next 15+/16 API split) **[Medium]**
- **Where:** `src/app/layout.tsx`
- **What:** No `export const viewport: Viewport`. Legacy embedded viewport in `metadata.viewport` is deprecated.
- **Why it matters:** Theme color, color scheme, initial scale all live here. Missing forfeits dark-mode browser chrome; produces console warning.
- **Fix:** Add a separate `viewport` export with `width: "device-width"`, `initialScale: 1`, dark `themeColor`, `colorScheme: "dark"`.

### 2.2 Indexability

#### F-006 — No `robots.txt` or `robots.ts` **[Critical]**
- **Where:** Missing from `src/app/` and `public/`.
- **Why it matters:** Crawlers will crawl everything by default — including `/api/*`, `/ticket/add`, and dynamic per-ID pages — wasting crawl budget and potentially indexing private endpoints.
- **Fix:** Add `src/app/robots.ts` returning `MetadataRoute.Robots`. For an internal tool, disallow `/` entirely. For public deployment, disallow `/api/`, `/ticket/`, and reference the sitemap.

#### F-007 — No `sitemap.xml` or `sitemap.ts` **[High]**
- **Where:** Missing from `src/app/`.
- **Why it matters:** Sitemaps accelerate discovery and provide lastmod hints for re-crawl prioritization.
- **Fix:** Add `src/app/sitemap.ts`. Include `/`, `/tickets` statically. If tickets are public, fetch IDs from MongoDB and emit `/ticket/<id>` with `lastModified: ticket.updatedAt`.

#### F-008 — No meta robots / `X-Robots-Tag` for sensitive routes **[High]**
- **Where:** `/ticket/[mode]`, `/api/*`
- **Why it matters:** A search engine could index `/ticket/add` (a blank form) or surface a JSON API response, leaking schema.
- **Fix:** Export `metadata = { robots: { index: false, follow: false } }` on form/admin routes. For `/api/*`, set `X-Robots-Tag: noindex` via `headers()` in `next.config.ts`.

#### F-009 — No `headers()` in `next.config.ts` for security/SEO **[Medium]**
- **Where:** `next.config.ts` (empty)
- **What:** No HSTS, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`, or `X-Robots-Tag` configured.
- **Why it matters:** Google factors HTTPS/security headers into trust; missing HSTS is a Lighthouse Best-Practices hit.
- **Fix:** Add `headers()` returning HSTS, `X-Content-Type-Options: nosniff`, `Referrer-Policy: strict-origin-when-cross-origin`, and `X-Robots-Tag: noindex` on `/api/:path*`.

### 2.3 Structured Data

#### F-010 — No JSON-LD anywhere **[High]**
- **Where:** Entire app.
- **What:** No `Organization`, `WebSite`, `WebApplication`, `BreadcrumbList`, or `SoftwareApplication` schema.
- **Why it matters:** Structured data qualifies content for rich results, knowledge panels, and AI search citations.
- **Fix:** Inject `<script type="application/ld+json">` in root layout for `Organization` + `WebSite` (with `SearchAction` if header search becomes a real route). Add `BreadcrumbList` to listing pages.

#### F-011 — Header search input is non-functional and non-indexable **[Medium]**
- **Where:** `src/components/header/index.tsx` lines 14–18
- **What:** Plain `<input placeholder="Ticket ara...">` with no form, no action, no `name`, no submit handler.
- **Why it matters:** Cannot back a `WebSite` `SearchAction` schema; users get no actual search.
- **Fix:** Wrap in `<form action="/search" method="GET">`, give input `name="q"`, add `WebSite` JSON-LD with `potentialAction` pointing to `/search?q={search_term_string}`.

### 2.4 Performance / CWV signals

#### F-012 — Empty `next.config.ts` — no production optimizations **[High]**
- **Where:** `next.config.ts`
- **What:** No `images.remotePatterns`, no `poweredByHeader: false`, no caching headers for `/_next/static`.
- **Why it matters:** Future remote images will skip optimization (LCP killer). `X-Powered-By: Next.js` leaks stack info.
- **Fix:** Set `poweredByHeader: false`, define `images.remotePatterns`/`formats: ["image/avif","image/webp"]`, add `Cache-Control: public, max-age=31536000, immutable` for `/_next/static/:path*`.

#### F-013 — Two `getStatistics()` DB calls per request **[High]**
- **Where:** `src/app/page.tsx` line 7 calls `getStatistics()`; `src/components/header/statistics.tsx` line 5 also calls it. `<Header />` is in the root layout, so `/` triggers two parallel identical DB round-trips on every dashboard navigation.
- **Why it matters:** Doubled DB latency inflates **TTFB** and therefore **LCP**. CWV is a ranking factor.
- **Fix:** Wrap `getStatistics` with React's `cache()` to dedupe within a render. Optionally use `unstable_cache` with a 30s TTL since stats need not be real-time.

#### F-014 — No font display strategy / preload control **[Medium]**
- **Where:** `src/app/layout.tsx` lines 7–15
- **What:** Geist + Geist_Mono loaded with defaults; `display` not set; mono font preloaded though used in only one place.
- **Why it matters:** Blocking render hurts LCP; preloading the rarely-used mono is wasted bandwidth.
- **Fix:** Set `display: "swap"` on both, `preload: false` on `Geist_Mono`.

#### F-015 — `globals.css` overrides Geist with Arial **[Medium]**
- **Where:** `src/app/globals.css` line 25 — `body { font-family: Arial, Helvetica, sans-serif; }`
- **What:** This `body` rule defeats the Geist font loaded via `next/font` and assigned to `--font-geist-sans`.
- **Why it matters:** You pay the network cost of Geist but never display it. Wasted bytes; potential CLS if Geist ever does apply.
- **Fix:** Remove the rule, or change to `font-family: var(--font-geist-sans), Arial, sans-serif;`.

#### F-016 — Sidebar is a client component for one boolean **[Low]**
- **Where:** `src/components/sidebar/index.tsx` line 1 — `"use client"`
- **What:** Whole sidebar (logo, nav, buttons) ships as client JS to manage one collapse boolean.
- **Why it matters:** Larger client bundle, worse INP/TBT on mobile.
- **Fix:** Extract only the collapse toggle/container width into a tiny client island; keep nav `<Link>` list as server component passed via children.

#### F-017 — `chart.js` + `react-chartjs-2` may not be code-split per route **[Medium]**
- **Where:** `src/components/home/doughnut-chart` (used only on `/`)
- **Why it matters:** If sync-imported, ~70KB gz of chart code can leak into other routes.
- **Fix:** Verify with bundle analyzer; if needed wrap chart components with `next/dynamic` (`ssr: false`).

### 2.5 Accessibility-as-SEO

#### F-018 — `<html lang="en">` but Turkish content **[Critical]**
- **Where:** `src/app/layout.tsx` line 28
- **What:** Document declares English; UI strings ("Dashboard", "Kategori Dağılımı", "Yeni", "Kullanıcı", "Çıkış", "Oluşturuldu", etc.) are all Turkish.
- **Why it matters:** Among the worst SEO mistakes possible. Confuses Google's language detection; page may be excluded from Turkish SERPs and mistranslated by Chrome auto-translate; screen readers will read Turkish text in English voice; breaks any future hreflang strategy.
- **Fix:** Change `lang="en"` to `lang="tr"`. If multilingual is planned, structure under `[locale]` segments and set `lang` dynamically.

#### F-019 — Multiple `<h1>` per page **[Medium]**
- **Where:** Sidebar contains `<h1>Rudder</h1>` (`src/components/sidebar/index.tsx` line 24), rendered on every page in addition to each route's own `<h1>`.
- **Why it matters:** Two `<h1>` dilutes topical signal; accessibility/SEO smell.
- **Fix:** Change the sidebar logo `<h1>` to `<span>` or `<p>` (it's decorative branding, not the primary page heading).

#### F-020 — Icon-only buttons missing `aria-label` **[High]**
- **Where:**
  - `src/components/header/index.tsx` lines 24–34 (Bell, Mail, Settings, profile)
  - `src/components/sidebar/index.tsx` lines 27–32 (collapse), 52–63 (Help/Logout when collapsed)
  - `src/components/tickets/delete-button.tsx` line 26 (Trash)
- **Why it matters:** Screen readers announce nothing; Lighthouse a11y score drops; a11y is increasingly tied to SEO.
- **Fix:** Add `aria-label="Bildirimler"`, `"Mesajlar"`, `"Ayarlar"`, `"Menüyü daralt/genişlet"`, `"Sil"`, etc.

#### F-021 — Search input has no associated label **[Medium]**
- **Where:** `src/components/header/index.tsx` lines 14–18
- **Why it matters:** Placeholder is not an accessible name; screen readers announce "edit text" with no context.
- **Fix:** Add `aria-label="Ticket ara"` or wrap with a visually hidden `<label>`.

#### F-022 — Form inputs lack proper label association **[Medium]**
- **Where:** `src/components/form/index.tsx` and `src/components/form/field.tsx`
- **What:** Inputs for title/description/category/status have no `id` props, so `<Field label>` cannot wire `htmlFor`. Only the radios use `id={priority}`.
- **Why it matters:** Clicking labels won't focus inputs; screen readers can't announce field names.
- **Fix:** Give each input a stable `id`; have `Field` render `<label htmlFor={id}>`.

#### F-023 — Decorative icons not hidden from AT **[Low]**
- **Where:** All `lucide-react` icons throughout
- **Why it matters:** Screen readers may announce decorative imagery beside its text label, creating noise.
- **Fix:** Add `aria-hidden="true"` to icons that sit beside textual labels.

#### F-024 — Logo `alt="logo"` is non-descriptive **[Low]**
- **Where:** `src/components/sidebar/index.tsx` line 23
- **Why it matters:** "logo" provides zero information.
- **Fix:** Use `alt="Rudder"` or `alt=""` if brand text appears beside it.

### 2.6 Social / Open Graph / Twitter

#### F-025 — No Open Graph tags **[High]**
- **Where:** `src/app/layout.tsx` metadata
- **Why it matters:** Any link shared in Slack/WhatsApp/LinkedIn/Discord/Twitter/iMessage renders as a bare URL with no preview.
- **Fix:** Add `openGraph: { type: "website", siteName: "Rudder", locale: "tr_TR", images: [...] }`.

#### F-026 — No `opengraph-image` file **[High]**
- **Where:** No `src/app/opengraph-image.{png,jpg,tsx}` or `twitter-image.*`.
- **Why it matters:** Even with OG tags, no image = ugly low-CTR previews.
- **Fix:** Add `src/app/opengraph-image.tsx` (using `ImageResponse` for dynamic 1200x630) and `src/app/twitter-image.tsx`.

#### F-027 — No Twitter Card metadata **[Medium]**
- **Where:** Root metadata
- **Why it matters:** X/Twitter falls back to small `summary` thumbnail.
- **Fix:** Add `twitter: { card: "summary_large_image", title, description, images, creator: "@..." }`.

### 2.7 Internationalization

#### F-028 — No i18n routing or hreflang **[Medium]**
- **What:** Content hard-coded in Turkish; no `[locale]` segment, no `next-intl`, no `alternates.languages`.
- **Why it matters:** Lack of routing convention forces a costly future refactor; immediate damage is the `lang` mismatch (F-018).
- **Fix:** Even single-locale, declare `openGraph.locale: "tr_TR"` and `alternates.languages: { "tr-TR": "/" }`. Long-term, restructure under `app/[locale]/...`.

#### F-029 — Geist font subset only `latin` for Turkish content **[Low]**
- **Where:** `src/app/layout.tsx` lines 7–15 — `subsets: ["latin"]`
- **Why it matters:** `ç ğ ı İ ö ş ü` may fall back inconsistently across font versions, causing subtle CLS.
- **Fix:** Use `subsets: ["latin", "latin-ext"]` for both Geist instances.

### 2.8 Mobile / PWA

#### F-030 — No `manifest.ts` / `manifest.json` **[Medium]**
- **Where:** Missing from `src/app/` and `public/`.
- **Why it matters:** No "Add to Home Screen", no app icons, no theme/background color, no PWA installability — valuable for a daily-use ops dashboard.
- **Fix:** Add `src/app/manifest.ts` returning `MetadataRoute.Manifest` with `name`, `short_name: "Rudder"`, `start_url: "/"`, `display: "standalone"`, `background_color: "#0a0a0a"`, `theme_color: "#18181b"`, and 192/512 icons.

#### F-031 — No favicon variants beyond default **[Low]**
- **Where:** `src/app/`
- **What:** No `icon.tsx`/`icon.png` (16, 32, 192, 512), no `apple-icon.png` (180x180).
- **Fix:** Add `src/app/icon.png` (or `icon.tsx`) and `src/app/apple-icon.png`. Next.js auto-wires the link tags.

#### F-032 — Viewport / theme-color not declared **[Medium]**
- **Where:** `src/app/layout.tsx`
- **Fix:** Export `viewport: { themeColor: [{ media: "(prefers-color-scheme: dark)", color: "#0a0a0a" }] }`.

### 2.10 URL Structure

#### F-035 — `/ticket/[mode]` overloads one segment for two intents **[Medium]**
- **Where:** `src/app/ticket/[mode]/page.tsx`
- **What:** Same path handles `mode = "add"` (create) and `mode = "<ObjectId>"` (edit), distinguished by string comparison.
- **Why it matters:** Mixing verbs and resource IDs complicates canonical/sitemap logic, breaks analytics URL grouping, and risks indexing stale ID URLs.
- **Fix:** Restructure as `/tickets/new` for create and `/tickets/[id]/edit` for edit.

#### F-036 — Singular vs plural inconsistency: `/tickets` vs `/ticket/[id]` **[Low]**
- **Why it matters:** Internal-link clarity and crawl pattern recognition prefer consistency.
- **Fix:** Standardize on `/tickets`, `/tickets/[id]`, `/tickets/new`, `/tickets/[id]/edit`.

#### F-037 — No trailing-slash policy declared **[Low]**
- **Where:** `next.config.ts`
- **Fix:** Explicitly set `trailingSlash: false`; ensure canonicals match.

## 4. Strategic Recommendation

Because this is a **ticket management dashboard** — an internal/operational tool — the realistic SEO strategy is **defensive, not offensive**:

1. **Index nothing application-side.** Ship `robots.ts` with `Disallow: /` and `metadata.robots = { index: false, follow: false }` in root layout.
2. **Treat any marketing site as a separate Next.js project** under the apex domain (e.g., `rudder.app`) with full SEO investment, while the dashboard lives at `app.rudder.app` and is fully de-indexed.
3. **Still execute Phases 1 and 2** even for an internal tool: branded metadata, correct `lang`, OG cards (so Slack/iMessage previews look right), security headers, and a manifest are quality investments independent of SEO.
4. **Performance work (F-013, F-014, F-015) pays back regardless of indexing intent** — every dashboard user benefits from faster TTFB/LCP.

If the intent is a **public, multi-tenant SaaS** with marketing surfaces, then Phase 1–3 becomes mandatory and the route restructure (F-035) should happen **before** any URLs accumulate inbound links.
